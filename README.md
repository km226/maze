# SDL2 2D 미로 탐색 게임

SDL2 라이브러리를 활용하여 구현한 2차원 미로 탐색 게임입니다.
프로그램은 무작위로 생성된 미로에서 플레이어가 시작점에서 출발해 도착점까지 이동하는 과정을 시각적으로 표현하며, 사용자는 키보드 입력으로 플레이어를 조작할 수 있습니다.
시야가 제한된 상태에서 미로를 탐색하여 길을 찾도록 했고, 단순한 콘솔 출력이 아닌 그래픽 기반 인터페이스(SDL2)로 몰입도를 높이고 플레이어를 이미지로 표현해 시각적 완성도를 높였습니다.

## 주요 기능

- **미로 자동 생성**
  - 2차원 배열을 이용해 미로를 구성
  - 시작점부터 도착점까지 반드시 이동 가능한 경로 생성
  - 프로그램 실행 시마다 랜덤하게 미로 생성
- **플레이어 이동**
  - 키보드 입력(W/A/S/D)으로 이동
  - 벽 충돌 시 이동 제한
  - 배열 범위를 벗어나는 이동 방지
- **부분 시야 시스템**
  - 플레이어 주변 일정 범위만 시야 확보
  - 제한된 시야에서 길을 찾는 탐험형 게임 느낌 강화
- **그래픽 출력 및 플레이어 이미지 출력**
  - 미로를 격자 형태로 시각화
  - 벽, 길, 도착점을 각각 다른 색상으로 구분
  - SDL_image 라이브러리로 PNG 이미지 로드
- **도착 판정 및 종료**
  - 플레이어가 도착점에 도달하면 게임 종료
  - 일정 시간(2초) 동안 전체 맵 공개 후 프로그램 종료

## 코드 구조

원본은 단일 파일로 작성되었던 코드를 기능 단위로 분리했습니다.

```
src/
├── maze.h    # 상수, 전역 변수 선언, 함수 프로토타입
├── maze.c    # 미로 생성 / 시야 갱신 / 플레이어 이동 / 전체 공개 로직
├── render.c  # SDL 화면 렌더링
└── main.c    # SDL 초기화, 이벤트 루프, 게임 종료 처리
```

| 파일 | 역할 |
| --- | --- |
| `maze.h` | `N`, `CELL`, `WALL`, `ROAD` 등 상수 정의와 `maze`, `visible`, `px/py`, `ex/ey` 전역 변수 선언, 각 소스 파일의 함수 프로토타입 선언 |
| `maze.c` | `generate_maze()`로 무작위 미로 생성, `update_visibility()`로 플레이어 주변 원형 시야 계산, `move_player()`로 이동 처리, `reveal_all()`로 클리어 시 전체 맵 공개 |
| `render.c` | `render_maze()`로 미로 격자·벽·도착점·플레이어 이미지를 렌더러에 그리는 역할 |
| `main.c` | SDL/SDL_image 초기화, 창·렌더러 생성, 키 입력 이벤트 루프, 도착 판정 및 종료 처리 |

## 빌드 방법

SDL2, SDL2_image 개발 패키지가 설치되어 있어야 합니다.

```bash
# Ubuntu / WSL 예시
sudo apt install libsdl2-dev libsdl2-image-dev

gcc -o maze src/main.c src/maze.c src/render.c -lSDL2 -lSDL2_image
./maze
```

실행 파일과 같은 경로에 플레이어로 사용할 `image.png` 파일이 있어야 합니다.

## 조작법

| 키 | 동작 |
| --- | --- |
| `W` | 위로 이동 |
| `A` | 왼쪽으로 이동 |
| `S` | 아래로 이동 |
| `D` | 오른쪽으로 이동 |

도착점(초록색 칸)에 도달하면 전체 미로가 공개되고 2초 뒤 프로그램이 종료됩니다.

## 구현 후기 및 결론

이번 프로젝트를 통해 조건문과 반복문, 배열 활용 능력을 키울 수 있었고, 각 기능별로 함수를 분리하는 설계의 중요성을 체감했습니다.
콘솔에서 동작하는 것이 아니라 실제 게임처럼 실행할 수 있도록 SDL 외부 라이브러리를 활용하는 방법을 연습할 수 있었습니다.

다만 아쉬운 점으로는 난이도 조절 기능이 없다는 점, 그리고 시작부터 도착까지 항상 오른쪽/아래로만 이동하는 경로 생성 방식 때문에 매번 비슷한 구조로만 길이 만들어져 실제 미로보다는 단순한 길찾기 게임에 더 가깝다는 점이 있습니다.

## 향후 개선 계획
